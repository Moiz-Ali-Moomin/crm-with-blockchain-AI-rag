Provision the Hetzner Server

Log in to console.hetzner.cloud → New Project → Add Server. Use the following configuration:

FieldValue
LocationNuremberg (EU) or Ashburn (US)
ImageUbuntu 22.04 LTS
Type — MinimumCX22 — 2 vCPU, 4 GB RAM · ~€4.51/mo
Type — RecommendedCX32 — 4 vCPU, 8 GB RAM · ~€9.28/mo
SSH KeyAdd your public key (~/.ssh/id_rsa.pub)
FirewallCreate new (see rules below)

Hetzner Firewall Rules

In the Hetzner console under Firewalls, create these inbound rules:

DirectionProtocolPortSource
InboundTCP22Your IP only
InboundTCP80Any (0.0.0.0/0)
InboundTCP443Any (0.0.0.0/0)
OutboundAnyAnyAny
⚠

Restrict port 22 to your own IP address. This prevents brute-force SSH attacks. If your IP changes, update the rule in the Hetzner console.

2

Point Your Domain to the Server

In your domain registrar's DNS settings, create two A records pointing to your Hetzner server IP:

TypeNameValueTTL
A@YOUR_HETZNER_IP300
AwwwYOUR_HETZNER_IP300

Verify DNS propagation (wait 5–30 min):

bash 
ping your-domain.com
# Should resolve to your Hetzner IP

nslookup your-domain.com
3

Initial Server Setup

SSH into the server

bash 
ssh root@YOUR_HETZNER_IP

Update system packages

bash 
apt update && apt upgrade -y

Create a non-root deploy user

bash 
adduser deploy
usermod -aG sudo deploy

# Copy SSH key to deploy user
mkdir -p /home/deploy/.ssh
cp ~/.ssh/authorized_keys /home/deploy/.ssh/
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys

Harden SSH — disable root login & passwords

bash 
nano /etc/ssh/sshd_config

# Set these values:
PermitRootLogin         no
PasswordAuthentication  no

systemctl restart sshd

Enable UFW firewall

bash 
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
ufw status
ℹ

From this point, SSH as the deploy user — not root. All commands below assume you are logged in as deploy.

bash 
# Exit root session and reconnect as deploy
exit
ssh deploy@YOUR_HETZNER_IP

Install Docker

bash 
# Install Docker Engine (official script)
curl -fsSL https://get.docker.com | sh

# Add deploy user to docker group (no sudo needed)

# Verify

Configure Environment Variables

bash 
cp crm-backend/.env.example crm-backend/.env
nano crm-backend/.env

Generate strong JWT secrets

bash — run twice 
openssl rand -hex 32
# Copy output → JWT_SECRET

openssl rand -hex 32
# Copy output → JWT_REFRESH_SECRET

Production .env values

crm-backend/.env

NODE_ENV=production
PORT=3001

# ── SMS / WhatsApp ────────────────────────────
TWILIO_ACCOUNT_SID=ACxxx
TWILIO_AUTH_TOKEN=xxx
TWILIO_PHONE_NUMBER=+1234567890

# ── Payments ──────────────────────────────────
STRIPE_SECRET_KEY=sk_live_xxx

# ── AI / RAG ──────────────────────────────────

# ── Blockchain ────────────────────────────────

⛔

Use container hostnames (postgres, redis, mongodb) in DATABASE_URL/REDIS_URL/MONGO_URI — not localhost. Inside Docker, containers talk to each other by service name.

7

Configure Nginx & Get SSL

Update Nginx config with your domain

bash 
nano /opt/nexus-crm/nginx/conf.d/app.conf

# Replace ALL occurrences of "your-domain.com" with your actual domain

Start Nginx on HTTP first (needed for SSL challenge)

Temporarily comment out the HTTPS server block — only keep the HTTP block active. Then:

bash 
cd /opt/nexus-crm
docker compose -f docker-compose.prod.yml up -d nginx
8

Issue SSL Certificate

bash 
docker compose -f docker-compose.prod.yml run --rm certbot certonly \
  --webroot \
  --webroot-path=/var/www/certbot \
  -d your-domain.com \
  -d www.your-domain.com \
  --email your@email.com \
  --agree-tos \
  --no-eff-email

After the certificate is issued, re-enable the HTTPS block in nginx/conf.d/app.conf, then reload:

bash 
# Re-enable HTTPS server block in nginx/conf.d/app.conf
nano nginx/conf.d/app.conf

# Reload Nginx with new config
docker compose -f docker-compose.prod.yml restart nginx

12

Automated Backups

Create the backup script

bash 
sudo mkdir -p /opt/backups/postgres
sudo nano /opt/backups/backup-db.sh
/opt/backups/backup-db.sh 
#!/bin/bash
set -e

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/opt/backups/postgres

mkdir -p $BACKUP_DIR

docker compose -f /opt/nexus-crm/docker-compose.prod.yml exec -T postgres \
  pg_dump -U crm_user crm_db | gzip > "$BACKUP_DIR/crm_db_$TIMESTAMP.sql.gz"

# Keep 30 days of backups
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

echo "Backup complete: crm_db_$TIMESTAMP.sql.gz"
bash 
sudo chmod +x /opt/backups/backup-db.sh

# Test it manually
sudo /opt/backups/backup-db.sh

# Set up cron — runs at 2am daily
crontab -e
crontab 
# Daily DB backup at 2am
0 2 * * * /opt/backups/backup-db.sh >> /var/log/crm-backup.log 2>&1

# SSL certificate renewal every Monday at 3am
0 3 * * 1 cd /opt/nexus-crm && docker compose -f docker-compose.prod.yml run --rm certbot renew && docker compose -f docker-compose.prod.yml restart nginx >> /var/log/certbot-renew.log 2>&1

Restore from backup

bash 
gunzip -c /opt/backups/postgres/crm_db_20260401_020000.sql.gz | \
  docker compose -f docker-compose.prod.yml exec -T postgres \
  psql -U crm_user -d crm_db

Monitoring & Useful Commands

Container status & resource usage

bash 
# All container statuses
docker compose -f docker-compose.prod.yml ps

# Live resource usage (CPU, memory)
docker stats --no-stream

# Health check endpoint
curl https://your-domain.com/api/health | python3 -m json.tool

Logs

bash 
# All services
docker compose -f docker-compose.prod.yml logs -f

# Single service
docker compose -f docker-compose.prod.yml logs -f api
docker compose -f docker-compose.prod.yml logs -f web
docker compose -f docker-compose.prod.yml logs -f nginx

# Workers only
docker compose -f docker-compose.prod.yml logs -f api | grep -E "Worker|Queue|job"

Database access

bash 
# PostgreSQL shell
docker compose -f docker-compose.prod.yml exec postgres psql -U crm_user -d crm_db

# Redis CLI
docker compose -f docker-compose.prod.yml exec redis redis-cli ping
docker compose -f docker-compose.prod.yml exec redis redis-cli info memory

# Enter API container
docker compose -f docker-compose.prod.yml exec api sh

Restart a single service

bash 
docker compose -f docker-compose.prod.yml restart api
docker compose -f docker-compose.prod.yml restart nginx

Disk usage

bash 
# Docker volumes and images
docker system df

# Clean up unused images (safe)
docker image prune -f

Final Deployment Checklist

  • Hetzner CX22+ server provisioned (Ubuntu 22.04)
  • DNS A records point to server IP
  • Non-root deploy user created
  • Root SSH login disabled, password auth disabled
  • UFW firewall enabled (22, 80, 443 only)
  • Docker + Docker Compose installed
  • Repo cloned to /opt/nexus-crm
  • crm-backend/.env filled with production values
  • JWT secrets generated with openssl rand -hex 32
  • Database URLs use container hostnames (not localhost)
  • nginx/conf.d/app.conf updated with real domain
  • SSL certificate issued via Certbot
  • HTTPS server block re-enabled in Nginx config
  • docker compose -f docker-compose.prod.yml up -d --build
  • All containers show healthy in docker compose ps
  • prisma migrate deploy run successfully
  • Demo seed run (first time)
  • Health endpoint returns {"status":"ok"}
  • GitHub Actions secrets configured (11 server + app secrets)
  • CD pipeline triggered successfully on push to main
  • Daily backup cron configured
  • SSL renewal cron configured
🚀

When all boxes are checked, your production CRM is fully deployed, secured, and automated. Every git push to main deploys automatically.
